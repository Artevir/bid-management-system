# 🎯 标书全流程管理平台

> 企业级投标全流程管理系统，支持项目创建、文档编制、团队协作、审核签发、导出投标等完整业务流程

---

## 📊 项目概览

- **总任务数**：82个
- **P0任务**：52个（必须首发）
- **P1任务**：24个（首版增强）
- **P2任务**：6个（后续迭代）
- **总工期**：15周
- **团队规模**：9人

---

## 📅 4个开发阶段

### 阶段1：基础设施与核心底座（Week 1-4）

- ✅ 权限与安全底座（8个任务）
- ✅ 项目管理与工作台（6个任务）
- ✅ 文件接入与存储中心（6个任务）

### 阶段2：文档智能与知识中台（Week 5-8）

- ✅ OCR与文档结构化抽取（10个任务）
- ✅ 投标响应矩阵（7个任务）
- ✅ 企业投标知识中台（8个任务）

### 阶段3：智能编标与审核流程（Week 9-12）

- ✅ 智能编标与在线编辑（8个任务）
- ✅ 智能审校与合规（7个任务）
- ✅ 四级审核流与签发控制（7个任务）

### 阶段4：导出适配与治理优化（Week 13-15）

- ✅ 导出与电子投标适配（5个任务）
- ✅ 项目驾驶舱与运营分析（4个任务）
- ✅ AI治理、评测与观测（6个任务）

## 🧪 测试

### E2E端到端测试

项目采用 Playwright 进行端到端测试，覆盖核心业务流程。

```bash
# 安装浏览器
npx playwright install

# 运行测试
pnpm test:e2e

# UI模式调试
pnpm test:e2e:ui

# 查看报告
pnpm test:e2e:report
```

详细测试指南请参阅 [E2E测试指南](docs/e2e-testing-guide.md)。
