# 上线检查清单（值班版）

## 1. 适用范围

用于生产发布窗口的值班人员快速执行与判定，覆盖：

- 发布前检查
- 发布中灰度观察
- 发布后冒烟验证
- 异常回滚处置

## 2. 发布前（T-30 分钟）

### 必做命令

```bash
pnpm build
pnpm release:gate
```

### 通过标准

- `pnpm build` 退出码为 0
- `pnpm release:gate` 全部检查项通过
- 无新增高优先级告警（认证、静态资源、数据库连接）

### 失败处理

- 任一失败：停止发布，记录失败项，通知研发负责人修复后重试

## 3. 发布中（灰度 10-15 分钟）

### 关键观测点

- 登录链路：未登录访问受保护页应跳转 `/login`
- 登录跳转：登录后可按 `next` 回到目标页面
- 静态资源：无 `Loading chunk failed`
- 核心项目接口：
  - `GET /api/projects`
  - `GET /api/projects/{id}`
  - `GET /api/projects/{id}/phases`
  - `GET /api/projects/{id}/milestones`

### 接口契约快速判定

- 返回体包含 `success`
- 业务数据位于 `data`
- 列表接口为 `data.items` + `data.total`

## 4. 发布后（T+15 分钟冒烟）

### 页面级验证

- 项目列表页：列表可加载、筛选正常
- 项目详情页：基础信息正常展示
- 甘特图页：阶段与里程碑可正常渲染
- 日历页：项目事件可正常生成
- 开标页/保证金页：项目下拉可正常加载

### 权限与安全验证

- 未登录访问 `/api/projects` 返回 401
- 管理员接口非管理员不可访问
- 回收站与解析任务只可访问本人数据

## 5. 回滚触发条件（任一满足立即回滚）

- 登录或鉴权错误率持续升高且 10 分钟内无法恢复
- `/_next/static/` 资源错误明显升高并影响主要页面可用性
- 核心 API 连续失败（`5xx`）影响主流程

## 6. 回滚动作（值班执行）

1. 回切上一稳定版本（镜像或构建目录）
2. 清理静态资源缓存（CDN/反向代理）
3. 复测 `/login`、`/api/health`、项目核心页面
4. 在值班群同步“已回滚 + 原因 + 后续修复负责人”

## 7. 交付记录要求

- 记录发布批次、执行人、开始/结束时间
- 记录灰度观察结果与异常
- 记录回滚与否及最终状态
