# Chunk加载与登录拦截修复记录-2026-04-14-V1.0

## 工作背景
- 线上出现 `Loading chunk 7177 failed`，报错指向 `/_next/static/chunks/app/layout-*.js`。
- 在其他浏览器中出现布局错乱，并且未登录状态可直接进入工作台。

## 问题分析
- 当前页面 HTML 引用了 `app/layout-*.js`，但对应资源请求返回 `400 Bad Request`，导致前端分包加载失败。
- 前端页面路由缺少强制登录守卫，未登录访问 `/` 时会直接渲染工作台外壳。
- 工作台页面接口请求失败后被兜底处理为空数据，视觉上容易误判为“已登录进入系统”。

## 本次修复内容
- 修改 `src/middleware.ts`，新增页面级登录守卫：
  - 未登录访问业务页面时重定向到 `/login`。
  - 重定向附带 `next` 参数，保留原访问路径与查询串。
  - 已登录访问 `/login` 时重定向回首页 `/`。
- 保持原有限流逻辑不变，仅将 API 判断复用为 `isApiRoute`。

## 影响范围
- 页面路由鉴权行为（非 `/api/*`）。
- 登录页与首页互跳逻辑。
- 不影响现有 API 权限中间件实现。

## 验证建议
- 无 `accessToken` 时访问 `/`，应自动跳转到 `/login`。
- 已登录后访问 `/login`，应自动跳转到 `/`。
- 线上部署后，确认 `/_next/static/chunks/app/layout-*.js` 返回 `200` 且类型为 JavaScript。

## 后续事项
- 排查部署流程中 `.next/static/chunks/app/*` 产物完整性。
- 排查 Nginx/CDN/WAF 是否对 `/_next/static/chunks/app/*` 存在拦截或异常回源规则。
