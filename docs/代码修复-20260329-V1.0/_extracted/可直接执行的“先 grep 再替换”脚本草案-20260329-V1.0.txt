使用方法
只扫描，不改
bash fix-pass.sh
扫描后直接替换“相对安全项”
DO_REPLACE=1 bash fix-pass.sh
替换后再自动构建
DO_REPLACE=1 DO_BUILD=1 bash fix-pass.sh
这份脚本当前自动处理了什么
自动处理 3 类：
1）res.data.xxx → res.xxx
这是你仓库里已经验证过的一类高频问题。
2）@/db/schema/audit-logs → @/lib/db/schema/audit-logs
这是你已经确认真实路径的一类。
3）import { cache } from '@/lib/cache' → import { redis as cache } from '@/lib/cache/redis-client'
这是你当前仓库里命中很多的旧导入写法。
这份脚本故意没自动处理什么
这些虽然常见，但不适合盲改：
1）request.ip
因为不同文件变量名和逻辑不一定一样。
2）Next.js 15 的 params
因为变量名有 id / roleId / tenantId / sessionId 等，不适合一刀切。
3）withDocumentPermission(...)(...)
因为每个 handler 写法不同，要人工改成：
const middleware = await withDocumentPermission(...);return middleware(request, ...);
4）llm.chat / llm.chatStream
因为你仓库里要统一走 invokeChat / streamChat，不能机械替换。
5）never[] / useState
这是类型设计问题，必须看上下文。
6）queryClient
需要补 useQueryClient()，不能自动插入。
