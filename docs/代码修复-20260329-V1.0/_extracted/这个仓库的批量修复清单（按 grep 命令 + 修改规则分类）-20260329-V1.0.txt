二、批量修复清单
A. Next.js 15 动态路由参数签名
这个你当前基本已经清掉了，但以后再遇到同类仓库可以直接这样扫。你前面这类问题已经出现过：动态路由 route.ts 第二个参数必须按 Promise 处理。
grep
grep -R "params }: { params: {" -n src/app/api
错误模式
{ params }: { params: { id: string } }const { id } = params
正确规则
{ params }: { params: Promise<{ id: string }> }const { id } = await params
适用对象
[id]/route.ts 
[roleId]/route.ts 
[tenantId]/route.ts 
[sessionId]/route.ts 
B. 中间件 async 返回函数调用残留
你已经抓到过典型问题：withDocumentPermission(...) 现在是 async function，不能再写成同步链式调用。
grep
grep -R "withDocumentPermission(" -n srcgrep -R "withChapterPermission(" -n srcgrep -R "withResourcePermission(" -n src
错误模式
return withDocumentPermission('read', () => documentId)(request, handler)
正确规则
const middleware = await withDocumentPermission('read', () => documentId)return middleware(request, handler)

其他所有 api/*/route.ts

统一规则
把所有中间件 handler 类型从：
Promise<NextResponse>
改成：
Promise<Response>
适用文件
src/lib/auth/middleware.ts 
修完后复查
