# 修复报告-20260403-V1.42

**版本**: V1.42  
**日期**: 2026-04-03

## 问题描述

构建失败，报错信息如下：

```
Type error: Argument of type 'Buffer<ArrayBufferLike>' is not assignable to parameter of type 'BodyInit | null | undefined'.
```

## 问题原因

在 Next.js 15 中，`NextResponse` 的类型定义更加严格。Node.js 的 `Buffer` 类型虽然在运行时可以正常工作，但在 TypeScript 类型检查时与 `NextResponse` 的 body 参数不兼容。

`xlsx` 库的 `XLSX.write()` 和 `docx` 库的 `Packer.toBuffer()` 都返回 `Buffer` 类型，这与 Next.js 15 的类型定义产生了冲突。

## 修复内容

**文件**: `src/app/api/interpretations/[id]/export/route.ts`

将生成函数的返回类型从 `Buffer` 改为 `Uint8Array`：

1. `generateExcel` 函数：
   - 修改前：`function generateExcel(...): Buffer`
   - 修改后：`function generateExcel(...): Uint8Array`
   - 返回值：`Buffer.from(XLSX.write(...)) as unknown as Uint8Array`

2. `generateWord` 函数：
   - 修改前：`async function generateWord(...): Promise<Buffer>`
   - 修改后：`async function generateWord(...): Promise<Uint8Array>`
   - 返回值：`Buffer.from(await Packer.toBuffer(doc)) as unknown as Uint8Array`

## 技术说明

- `Uint8Array` 是 Web 标准类型，`NextResponse` 兼容
- `Buffer` 是 Node.js 特有的类型，Next.js 15 在类型定义上更加严格
- 运行时 `Buffer` 可以正常工作，只是类型定义不匹配
- 使用类型断言 `as unknown as Uint8Array` 是安全且常用的做法
