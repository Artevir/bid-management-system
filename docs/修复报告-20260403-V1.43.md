# 修复报告-20260403-V1.43

**版本**: V1.43  
**日期**: 2026-04-03

## 问题描述

构建失败，报错信息如下：

```
Type error: Argument of type 'Uint8Array<ArrayBufferLike>' is not assignable to parameter of type 'BodyInit | null | undefined'.
```

## 问题原因

即使将返回类型改为 `Uint8Array`，Next.js 15 的 `NextResponse` 仍然无法正确识别。在 Next.js 15 / React 19 环境中，处理二进制文件下载的正确方式是使用标准的 Web `Response` API。

## 修复内容

**文件**: `src/app/api/interpretations/[id]/export/route.ts`

将 `new NextResponse(buffer, ...)` 改为 `new Response(buffer, ...)`：

修改前：
```typescript
return new NextResponse(buffer, {
  headers: { ... }
});
```

修改后：
```typescript
return new Response(buffer, {
  headers: { ... }
});
```

## 技术说明

- `Response` 是 Web 标准 API，Next.js 15 完全兼容
- `NextResponse` 是 Next.js 的封装，主要用于处理 JSON 和 URL 编码的数据
- 对于二进制文件（Excel、Word），使用标准 `Response` 更加可靠
- 两个 API 的接口基本一致，迁移成本很低
