# 修复报告-20260403-V1.45

**版本**: V1.45  
**日期**: 2026-04-03

## 问题描述

构建失败，报错信息如下：

```
Type error: This expression is not constructable.
Type 'typeof import("/www/wwwroot/bid-management-system/node_modules/@types/pdfmake/index")' has no construct signatures.

const pdfPrinter = new PdfPrinter(fonts);
```

## 问题原因

`pdfmake` 库的 TypeScript 类型定义与实际使用方式不匹配。`@types/pdfmake` 将库作为命名空间导出，而不是作为可构造的类。

## 修复内容

**文件**: `src/app/api/interpretations/[id]/export/route.ts`

修改前：
```typescript
import PdfPrinter from 'pdfmake';
const fonts = { ... };
const pdfPrinter = new PdfPrinter(fonts);
```

修改后：
```typescript
import PdfPrinter from 'pdfmake/build/pdfmake';
(PdfPrinter as any).setFonts({
  Helvetica: { ... },
});
const pdfPrinter = new PdfPrinter((PdfPrinter as any).getFonts());
```

## 技术说明

- `pdfmake/build/pdfmake` 是实际的入口点
- 使用 `setFonts()` 和 `getFonts()` 方法配置字体
- 使用类型断言 `as any` 绕过严格的类型检查
