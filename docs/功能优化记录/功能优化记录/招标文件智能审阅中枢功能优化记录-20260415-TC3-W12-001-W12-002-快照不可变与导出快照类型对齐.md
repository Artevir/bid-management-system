# 招标文件智能审阅中枢功能优化记录-20260415-TC3-W12-001-W12-002-快照不可变与导出快照类型对齐

## 1. 本次目标

- 补齐 `TC3-W12-001` 的核心约束：已发布快照不可修改，并提供可执行测试证明。
- 补齐 `TC3-W12-002` 的接口口径缺口：快照创建与导出接口对齐 `snapshotType/exportMode` 标准字段，避免“接口有名义参数但实现未生效”。

## 2. 本次代码改动

### 2.1 新增快照统一模型与不可变更规则

- 新增 `src/app/api/tender-center/_snapshot.ts`
  - 统一定义：
    - `snapshotType`：`requirements_snapshot/framework_snapshot/templates_snapshot/materials_snapshot/full_snapshot`
    - `exportMode`：`internal_consumption/downstream_module/manual_download/api_delivery`
    - `status`：`generated/published/invalidated`
  - 提供快照解析函数 `parseTenderSnapshotPayload`，兼容历史记录并补齐默认值。
  - 提供快照变更函数 `mutateTenderSnapshotPayload`，落实不可变规则：
    - 已发布快照若修改 `name/note` → 拒绝
    - 仅 `generated` 状态允许 `publish`
    - `invalidated` 状态禁止重复失效

### 2.2 生成快照接口补齐字段级校验

- 修改 `src/app/api/tender-center/projects/[projectId]/versions/[versionId]/snapshots/route.ts`
  - `POST` 新增 `snapshotType`、`exportMode` 入参校验（非法值返回 400）。
  - 快照写入内容新增结构化字段：
    - `snapshotType`
    - `exportMode`
    - `status=generated`
    - `publishedAt/publishedBy`
    - `invalidatedAt/invalidatedBy`
  - 列表接口读取快照时统一走标准解析，保证返回口径稳定。

### 2.3 快照详情接口补齐权限与变更能力

- 修改（并纳管）`src/app/api/tender-center/snapshots/[snapshotId]/route.ts`
  - `GET` 增加快照归属校验：仅上传者可访问对应快照，修复跨用户读取风险。
  - 新增 `PATCH`：
    - 支持 `action=publish|invalidate` 与 `name/note` 元数据更新
    - 应用不可变规则，违反规则返回 409
    - 落日志 `snapshot_updated`，保留审计痕迹

### 2.4 导出接口接入 snapshotType

- 修改 `src/app/api/tender-center/projects/[projectId]/versions/[versionId]/export/route.ts`
  - 新增 `snapshotType` 查询参数校验（非法值 400）。
  - 当传入 `snapshotType` 时，JSON 导出仅返回对应快照分支，避免客户端自行切分完整导出包。
