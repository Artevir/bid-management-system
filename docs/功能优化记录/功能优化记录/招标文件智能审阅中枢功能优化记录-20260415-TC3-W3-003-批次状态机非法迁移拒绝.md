# 招标文件智能审阅中枢功能优化记录-20260415-TC3-W3-003-批次状态机非法迁移拒绝

## 1. 本次目标

- 落地 `TC3-W3-003`：对批次状态（由 `interpretation.status` 承载）增加状态机约束。
- 对非法状态迁移进行代码级拒绝，并补齐单测矩阵，避免“跳状态”导致的数据不一致。

## 2. 代码改动

### 2.1 新增状态机模块

- 新增 `src/lib/interpretation/status-machine.ts`
  - 定义允许迁移：
    - `pending -> parsing`
    - `parsing -> completed|failed`
    - `failed -> pending|parsing`
  - 提供 `canTransitionInterpretationStatus` 与 `assertInterpretationStatusTransition`
  - 提供批次视图状态映射 `toTenderBatchStatus`
    - `pending -> pending`
    - `parsing -> running`
    - `completed -> succeeded`
    - `failed -> failed`

### 2.2 解析主流程接入状态机（拒绝非法迁移）

- 修改 `src/lib/interpretation/service.ts`
  - 新增 `transitionInterpretationStatus(...)`，以“当前状态 + 目标状态 + where 条件”原子更新，防并发冲突
  - `executeInterpretation` 改为严格迁移：
    - 启动：`pending/failed -> parsing`
    - 成功：`parsing -> completed`
    - 失败：仅在允许时迁移到 `failed`
  - `createInterpretation` 的重复上传路径增强：
    - `completed`：拒绝（保持原规则）
    - `parsing`：新增拒绝（避免解析中被覆盖）
    - `failed`：允许回到 `pending`（通过状态机迁移）

### 2.3 批次接口状态口径统一

- 修改以下接口返回的批次状态，统一为 `pending/running/succeeded/failed` 口径：
  - `src/app/api/tender-center/batches/[batchId]/route.ts`
  - `src/app/api/tender-center/batches/[batchId]/ai-tasks/route.ts`
  - `src/app/api/tender-center/projects/[projectId]/versions/[versionId]/batches/route.ts`

## 3. 单测矩阵

- 新增 `tests/unit/tender-center-batch-status-machine.test.ts`
  - 状态映射校验（4 条）
  - 合法迁移校验（5 条）
  - 非法迁移拒绝校验（5 条）
  - 断言抛错语义校验（1 条）
