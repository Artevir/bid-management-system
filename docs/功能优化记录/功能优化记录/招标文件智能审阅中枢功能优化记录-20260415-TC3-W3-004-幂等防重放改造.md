# 招标文件智能审阅中枢功能优化记录-20260415-TC3-W3-004-幂等防重放改造

## 1. 本次目标

- 面向 `TC3-W3-004`，在中枢核心写接口落地可执行的幂等防重放能力。
- 保证同一 `Idempotency-Key` 的重复提交不产生重复副作用（重复写日志/重复创建快照/重复触发流程）。

## 2. 本次代码改动

### 2.1 新增幂等工具

- 新增 `src/app/api/tender-center/_idempotency.ts`
  - `extractIdempotencyKey(headers)`：读取 `Idempotency-Key` / `x-idempotency-key`
  - `buildIdempotencyDigest(payload)`：稳定序列化后计算请求摘要
  - `lookupIdempotentResponse(...)`：基于 `bid_interpretation_logs` 查找历史幂等结果
  - `recordIdempotentResponse(...)`：写入幂等快照用于重放

### 2.2 接口能力增强（核心 POST）

- `src/app/api/tender-center/projects/[projectId]/versions/[versionId]/parse/route.ts`
  - 支持幂等键命中直接重放首次响应
  - 同键不同请求参数返回 `409`
- `src/app/api/tender-center/reviews/route.ts`
  - 复核任务创建支持幂等重放，避免重复副作用
- `src/app/api/tender-center/reviews/[reviewTaskId]/submit/route.ts`
  - 复核提交支持幂等重放，重复提交可直接返回首次结果
- `src/app/api/tender-center/projects/[projectId]/versions/[versionId]/snapshots/route.ts`
  - 快照创建支持幂等重放，避免重复快照写入

### 2.3 新增单测

- 新增 `tests/unit/tender-center-idempotency.test.ts`
  - 覆盖幂等键提取
  - 覆盖请求摘要稳定性（字段顺序不同摘要一致）
