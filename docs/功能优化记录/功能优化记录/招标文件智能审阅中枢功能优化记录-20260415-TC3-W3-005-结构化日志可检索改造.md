# 招标文件智能审阅中枢功能优化记录-20260415-TC3-W3-005-结构化日志可检索改造

## 1. 本次目标

- 按 `TC3-W3-005` 落地结构化日志字段：`taskId`、`batchId`、`traceId`。
- 将日志接入中枢核心写链路，确保至少一条可检索样例能被批次任务接口读到。

## 2. 本次代码改动

### 2.1 新增中枢结构化日志工具

- 新增 `src/app/api/tender-center/_trace.ts`
  - `getOrCreateTraceId(headers)`：优先读取 `x-trace-id/x-request-id/trace-id/traceparent`，否则生成新 traceId
  - `buildTenderTraceContext(...)`：构建标准化日志上下文（`taskId/batchId/traceId/event/timestamp`）
  - `logTenderTraceEvent(...)`：输出结构化控制台日志并落库到 `bid_interpretation_logs`（`operationType=trace_event`）

### 2.2 核心接口接入结构化日志

- `src/app/api/tender-center/projects/[projectId]/versions/[versionId]/parse/route.ts`
  - parse 触发时写入结构化日志
  - 响应中返回 `traceId/batchId/taskId`
- `src/app/api/tender-center/reviews/route.ts`
  - 复核任务创建时写入结构化日志
  - 响应中返回 `traceId/batchId`
- `src/app/api/tender-center/reviews/[reviewTaskId]/submit/route.ts`
  - 复核提交时写入结构化日志
  - 响应中返回 `traceId/batchId`
- `src/app/api/tender-center/projects/[projectId]/versions/[versionId]/snapshots/route.ts`
  - 快照创建时写入结构化日志
  - 响应中返回 `traceId/batchId`

### 2.3 可检索能力增强

- 修改 `src/app/api/tender-center/batches/[batchId]/ai-tasks/route.ts`
  - 解析日志 JSON 后回传 `taskId/batchId/traceId`
  - 对 `trace_event` 及历史日志均可兼容回读，支持批次维度检索观察

### 2.4 新增单测

- 新增 `tests/unit/tender-center-trace.test.ts`
  - Header 优先级提取
  - `traceparent` 解析
  - 结构化上下文生成校验
