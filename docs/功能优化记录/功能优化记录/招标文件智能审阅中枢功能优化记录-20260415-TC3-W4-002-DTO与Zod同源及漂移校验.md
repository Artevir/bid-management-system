# 招标文件智能审阅中枢功能优化记录-20260415-TC3-W4-002-DTO与Zod同源及漂移校验

## 1. 本次目标

- 落地 `W4-002`：实现 DTO 与 Zod 的同源生成，并在 CI 中增加可阻断的漂移校验。
- 避免接口字段变化后“类型与契约失配”无感漂移。

## 2. 本次代码改动

### 2.1 新增 DTO/Zod 同源生成脚本

- 新增 `scripts/generate-tender-center-dto-zod.py`
  - 以 `contracts/tender-center-w2-field-contract.json` 为单一源
  - 自动生成 `src/app/api/tender-center/_dto-contract.zod.ts`
  - 支持两种模式：
    - `--write`：更新生成文件
    - `--check`：比较现有生成文件与即时生成结果，不一致返回非 0（CI 阻断）

### 2.2 新增生成产物（Zod 契约映射）

- 新增 `src/app/api/tender-center/_dto-contract.zod.ts`
  - 覆盖 `tender-center` 已实现接口（method + path）
  - 为 `path/query/body` 生成 Zod 对象
  - 导出统一映射和 `z.infer` 类型，供后续接口校验或 SDK 类型复用

### 2.3 接入工程脚本与 CI 门禁

- 更新 `package.json`
  - `docs:v4-dto-zod:update`
  - `docs:v4-dto-zod:check`
- 更新 `.github/workflows/ci-cd.yml`
  - 在 `lint` job 增加 `W4-002 DTO/Zod 漂移校验（CI 阻断）`
