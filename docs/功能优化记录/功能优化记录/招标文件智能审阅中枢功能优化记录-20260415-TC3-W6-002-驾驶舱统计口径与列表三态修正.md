# 招标文件智能审阅中枢功能优化记录-20260415-TC3-W6-002-驾驶舱统计口径与列表三态修正

## 1. 本次目标

- 按 `TC3-W6-002` 补齐“首页/驾驶舱 API + 前端绑定”中的统计口径一致性，避免列表页分页导致首页卡片统计失真。
- 修正智能审阅首页列表三态渲染问题，确保表格场景使用统一的表格三态组件，避免错误的 DOM 结构和错误态丢失。

## 2. 本次代码改动

### 2.1 智能审阅 API 增加聚合统计口径

- 修改 `src/app/api/smart-review/route.ts`
  - 在列表查询之外新增聚合统计查询，返回：
    - `total`
    - `pendingReviewCount`
    - `parsedCount`
    - `approvedCount`
  - 统计与当前筛选条件同源，保证“列表数据”与“顶部统计卡片”口径一致，不再受单页数据数量影响。

### 2.2 智能审阅首页绑定真实统计并修复列表三态

- 修改 `src/app/smart-review/page.tsx`
  - 新增 `SmartReviewStats` 状态并绑定接口返回的 `stats`。
  - 顶部四个统计卡从“当前页 documents 本地过滤”切换为“后端聚合统计”：
    - 文档总数
    - 待审核
    - 已解析
    - 已通过
  - 列表区由错误的 `ListStateBlock`（`<div>`）渲染改为 `TableListStateRow`（`<tr>`）：
    - `loading`、`error`、`empty` 三态均可正确渲染
    - `error` 态支持重试（`onRetry=fetchDocuments`）
