# 招标文件智能审阅中枢功能优化记录-20260415-TC3-W7-002-批次推进UI60秒内联动更新

## 1. 本次目标

- 按 `TC3-W7-002` 落地“批次推进 UI 在 60s 内联动更新”。
- 采用轮询方案（15s 间隔）确保批次处于推进态时，列表页与详情页可自动刷新并在 60s 内反映状态变化。

## 2. 本次代码改动

### 2.1 解读列表页：由“手动触发后短轮询”升级为“运行中批次自动联动”

- 修改 `src/app/interpretations/page.tsx`
  - `fetchData` 增加 `silent` 模式，用于后台刷新时避免反复进入全屏 loading。
  - 移除原先 `pollStatus` 的“单条 2s 轮询 + 30s 超时”实现。
  - 新增自动联动轮询：当列表存在 `parsing` 记录时，每 `15s` 静默刷新一次列表与统计卡。
  - 启动解析成功后立即静默刷新一次，减少状态切换可见延迟。

### 2.2 智能审阅列表页：补齐运行中批次自动刷新

- 修改 `src/app/smart-review/page.tsx`
  - `fetchDocuments` 增加 `silent` 模式，后台刷新时不打断当前交互。
  - 当页面存在运行中记录时（`uploading/parsing/reviewing` 或 `reviewStatus=in_progress`），每 `15s` 自动刷新。
  - 通过 `keywordRef` 保持“点击搜索才生效”的既有交互，不改变原筛选行为。

### 2.3 智能审阅详情页：补齐推进态自动联动

- 修改 `src/app/smart-review/[id]/page.tsx`
  - `fetchDocument` 增加 `silent` 模式。
  - 详情页在推进态（`uploading/parsing/reviewing` 或 `in_progress`）下每 `15s` 自动刷新一次。
  - 使详情页与列表页在批次推进过程中保持一致，避免“列表已更新、详情仍旧状态”的不同步问题。
