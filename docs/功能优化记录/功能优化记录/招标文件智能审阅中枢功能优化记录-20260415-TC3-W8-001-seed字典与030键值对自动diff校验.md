# 招标文件智能审阅中枢功能优化记录-20260415-TC3-W8-001-seed字典与030键值对自动diff校验

## 1. 本次目标

- 按 `TC3-W8-001` 落地“seed 字典与 030 键值对 diff 自动核对”能力，形成可重复执行、可在 CI 阻断漂移的机制。
- 避免后续字典值在实现层与 030 上位口径发生无感漂移，保证状态/枚举标准可追溯。

## 2. 本次代码改动

### 2.1 新增 W8-001 自动校验脚本（支持 update/check 双模式）

- 新增 `scripts/check-v3-w8-seed-dictionary.py`
  - 从 `docs/招标文件智能审阅中枢/030...md` 自动提取字典键和值（限定在 5~12 章的正式状态/字典段）。
  - 与快照文件 `contracts/tender-center-w8-seed-dictionary.json` 做结构化 diff：
    - 缺失字典键（030 有 / seed 无）
    - 多余字典键（seed 有 / 030 无）
    - 字典值漂移（缺失值 / 多余值 / 顺序变化）
  - `--write`：更新快照并生成附录 I 报告。
  - `--check`：漂移即返回非 0，支持 CI 阻断。

### 2.2 落地 seed 字典快照与附录报告

- 新增 `contracts/tender-center-w8-seed-dictionary.json`
  - 作为 W8-001 的“可审计快照源”，固化 030 当前字典基线。
- 新增 `docs/招标文件智能审阅中枢开发计划-交付级别-20260415-V3.0-附录I-W8-001-seed字典与030键值对核对.md`
  - 自动输出统计指标、差异摘要和操作命令。

### 2.3 接入工程脚本与 CI 门禁

- 修改 `package.json`
  - 新增：
    - `docs:v3-w8-seed-dictionary:update`
    - `docs:v3-w8-seed-dictionary:check`
- 修改 `.github/workflows/ci-cd.yml`
  - 在 lint/type-check 门禁链中新增：
