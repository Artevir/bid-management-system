## 1. 本次目标

- 按 `000` / `010` 的核心主链路要求，优先落“项目版本 -> 原文分片 -> 要求资产”可见可操作能力。
- 避免继续做过程门禁，直接做系统内业务闭环页面和接口。

## 2. 本次代码改动

### 2.1 新增资产主链路提炼能力

- 新增 `src/lib/smart-review/assets.ts`
  - 将解析后的 `basicInfo/timeNodes/submissionRequirements/technicalSpecs/scoringItems/qualificationRequirements` 统一提炼为：
    - `segments`（原文分片视角）
    - `requirements`（要求资产视角）
  - 统一输出 `segmentId/source/confidence/isMandatory` 等治理字段，支持后续追溯和确认。

### 2.2 新增资产主链路 API

- 新增 `src/app/api/smart-review/[id]/assets/route.ts`
  - 提供主链路查询：项目信息、版本信息、分片列表、要求资产列表。
- 新增 `src/app/api/smart-review/[id]/assets/materialize/route.ts`
  - 支持把提炼后的要求资产入库到 `smart_response_matrix/smart_response_items`（矩阵名：`要求资产主链路`）。
  - 支持重复执行覆盖重建，保证可重入。
- 新增 `src/app/api/smart-review/[id]/assets/requirements/[itemId]/route.ts`
  - 支持单条要求状态更新（`pending/confirmed/rejected`），并回写矩阵统计。

### 2.3 增强解析后自动资产化

- 修改 `src/app/api/smart-review/[id]/parse/route.ts`
  - 在 AI 解析成功后，自动执行“要求资产主链路”入库：
    - 自动创建/复用主链路矩阵
    - 自动写入要求资产条目
    - 自动更新矩阵统计

### 2.4 新增资产主链路业务页面

- 新增 `src/app/smart-review/assets/page.tsx`
  - 提供“资产主链路台”前端可操作页面：
    - 选择文档版本
    - 一键入库要求资产
    - 查看原文分片列表
    - 查看要求资产列表
    - 对要求执行确认/驳回动作
  - 全量列表使用三态组件（loading/error/empty），满足全站一致性标准。

### 2.5 增加导航与入口

- 修改 `src/components/layout/app-layout.tsx`
  - 在“智能审阅中枢”菜单新增 `资产主链路台`（`/smart-review/assets`）。
- 修改 `src/app/smart-review/page.tsx`
  - 在中枢首页增加“资产主链路台”快捷入口按钮。
