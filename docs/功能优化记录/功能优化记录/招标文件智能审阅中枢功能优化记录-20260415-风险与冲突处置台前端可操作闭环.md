# 招标文件智能审阅中枢功能优化记录-20260415-风险与冲突处置台前端可操作闭环

## 1. 本次目标

- 依据 `000` 与 `010` 中“风险、冲突、人工复核可操作闭环”要求，补齐系统内可见可操作入口。
- 将已完成的 `tender-center` 风险/冲突后端 API 转化为前端可用功能，避免“只有接口没有业务页面”。

## 2. 本次代码改动

### 2.1 新增风险与冲突处置台页面

- 新增 `src/app/smart-review/governance/page.tsx`
  - 提供项目与版本选择器（调用 `GET /api/tender-center/projects` 与 `GET /api/tender-center/projects/{projectId}/versions`）。
  - 展示风险与冲突聚合统计（总数/高风险/中风险/冲突数）。
  - 风险列表支持 `关闭风险`（调用 `POST /api/tender-center/risks/{riskId}/close`）。
  - 冲突列表支持三种处理动作：
    - `采纳要求`（`resolutionType=accept_requirement`）
    - `采纳实际`（`resolutionType=accept_actual`）
    - `人工覆盖`（`resolutionType=manual_override`）
  - 所有动作完成后自动回刷列表，形成前端处置闭环。
  - 页面启用 `15s` 自动轮询，保证处置状态近实时联动。
  - 列表统一使用 `TableListStateRow` 三态（加载/错误/空）呈现。

### 2.2 补齐中枢导航入口

- 修改 `src/components/layout/app-layout.tsx`
  - 在“智能审阅中枢”菜单下新增 `风险与冲突处置台`，路径 `/smart-review/governance`。

### 2.3 强化中枢首页跳转入口

- 修改 `src/app/smart-review/page.tsx`
  - 在页面顶部新增“风险与冲突处置台”快捷按钮，直接进入治理页面执行操作。

## 3. 业务价值

- 风险与冲突从“后端可处理”升级为“前端可见、可操作、可回刷”。
- 让 `risk_item/conflict_item/review_task` 相关治理动作在系统中形成真实业务闭环。
- 对齐 `000/010` 对“资产中枢 + 治理闭环”的要求，而不是仅做过程性门禁脚本。
