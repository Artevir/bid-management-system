# 招标文件智能审阅中枢功能优化记录-20260415-风险与冲突治理闭环增强

## 1. 本次目标

- 直接增强系统内的中枢业务能力，不做演示脚本。
- 补齐“风险关闭/重开、冲突定案”的权限控制、状态回写和审计留痕，形成治理闭环。

## 2. 功能改动

### 2.1 冲突治理能力增强

- 更新 `src/app/api/tender-center/conflicts/[conflictId]/resolve/route.ts`
  - 新增 `resolutionType` 入参校验（`accept_requirement` / `accept_actual` / `manual_override`）。
  - 增加冲突存在性校验、关联版本存在性校验、上传者权限校验。
  - 增加已处理冲突重复处理拦截（409）。
  - 定案后回写 `actualValue`、`remarks`、`improvementSuggestion`、`checkedBy/checkedAt`。
  - 增加 `bidInterpretationLogs` 留痕（`conflict_resolved`）。

- 更新 `src/app/api/tender-center/conflicts/[conflictId]/route.ts`
  - 增加冲突详情读取权限校验（仅上传者可访问）。

### 2.2 风险治理能力增强

- 更新 `src/app/api/tender-center/risks/[riskId]/close/route.ts`
  - 将动作扩展为 `action=close|reopen`，支持风险关闭与重开。
  - 新增 action 参数合法性校验。
  - 对 `req-*` 与 `parse-*` 两类风险增加存在性校验、关联版本校验、上传者权限校验。
  - 对 `parse-*` 风险重开时强制要求说明（防止空重开）。
  - 写入状态回写和 `bidInterpretationLogs` 留痕（`risk_closed/risk_reopened/parse_risk_closed/parse_risk_reopened`）。

- 更新 `src/app/api/tender-center/risks/[riskId]/route.ts`
  - 增加风险详情读取权限校验。
  - 错误返回统一为 `tenderCenterError` 结构。

### 2.3 契约测试补充

- 更新 `tests/unit/tender-center-contract.test.ts`
  - 新增“无效 risk action 返回 400”断言。
  - 新增“无效 conflict resolutionType 返回 400”断言。
