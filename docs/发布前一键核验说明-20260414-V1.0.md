# 发布前一键核验说明（第四批后半段）

## 1. 目标

在发布前用统一命令快速校验关键发布前置条件，避免以下高频问题进入生产：

- 缺失核心环境变量导致运行时鉴权或定时任务失效
- Next.js 产物不完整导致启动失败或静态资源异常
- 人工检查遗漏导致发布质量不一致

## 2. 一键核验命令

```bash
pnpm release:gate
```

该命令对应脚本：`src/scripts/release-gate.ts`

## 3. 当前核验项

### 环境变量核验（必须）

- `DATABASE_URL`
- `JWT_SECRET`
- `CRON_SECRET`
- `E_SIGN_CALLBACK_SECRET`

### 构建产物核验（必须）

- `.next/standalone/server.js`
- `.next/static`

## 4. 建议发布顺序

1. 安装依赖：`pnpm install`
2. 执行构建：`pnpm build`
3. 执行核验：`pnpm release:gate`
4. 启动前冒烟：访问 `/login`、`/api/health`、核心业务页面
5. 通过后再执行部署发布

## 5. 灰度与回滚清单（建议）

### 灰度观察（首批流量）

- 登录链路：未登录重定向、登录后跳回 `next` 页面
- 静态资源：无 `Loading chunk failed`
- 关键 API：项目列表、项目详情、里程碑/阶段接口返回 `success/data` 契约

### 回滚触发条件

- 登录链路大面积 401/跳转异常
- `/_next/static/` 资源加载失败率持续升高
- 核心业务接口连续错误且 10 分钟内无法恢复

### 回滚动作

1. 回切到上一个稳定镜像或构建目录
2. 清理 CDN/反向代理缓存（如启用）
3. 验证 `/login` 与核心页面可用后恢复流量

## 6. 结果判定

- 核验脚本全部通过，判定为“可发布”
- 任一必检项失败，判定为“禁止发布”，需先修复后重跑
