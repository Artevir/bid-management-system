---

## 📋 项目概览

### 核心定位
**标书全流程管理平台 = AI-native生产平台**（而非单一文档管理工具）

### 建设目标
- 📄 **文档智能接入**：自动解析招标文件，结构化提取关键信息
- 🎯 **投标响应矩阵**：一键生成响应清单，精准匹配招标要求
- ✍️ **智能编标**：AI辅助首稿生成，知识库快速复用
- 🔍 **智能审校**：自动检测漏项、错项、格式问题
- ✔️ **多级审核**：四级审核流程，全程可追溯
- 📚 **企业知识中台**：历史标书、资质、案例结构化管理
- 🔐 **保密管控**：权限控制、审计日志、加密存储

### 核心原则
```
1. 先结构化，再生成
   ✅ 优先做识别、抽取、归档、校验与引用
   ❌ 避免一上来就无边界生成

2. 检索优先，生成受控
   ✅ 资质、案例、参数优先从知识库召回
   ✅ 仅在缺失时使用AI生成，且需人工确认

3. 规则兜底，AI增强
   ✅ 格式校验、时间节点、必填项用规则保证
   ✅ AI用于语义理解、相似召回、辅助审校

4. 过程可追溯，结果可审计
   ✅ 每个环节留痕，支持版本对比与责任定位
```

## 🛠️ 技术栈（AI-native 2.0）

### 核心技术栈

| 层级 | 技术选型 | 版本 | 说明 |
|------|----------|------|------|
| **框架** | Next.js | 15+ (App Router) | 全栈框架，支持SSR/SSG |
| **语言** | TypeScript | 5+ | 类型安全 |
| **前端** | React | 19+ | 最新特性 |
| **UI组件** | shadcn/ui | 最新 | Radix UI + Tailwind CSS |
| **样式** | Tailwind CSS | 4+ | 原子化CSS |
| **表单** | React Hook Form + Zod | 最新 | 类型安全的表单处理 |
| **数据库** | PostgreSQL | 15+ | 主数据库 |
| **ORM** | Drizzle ORM | 最新 | 类型安全ORM |
| **缓存** | Redis | 7+ | 会话缓存、热点数据 |
| **认证** | NextAuth.js | 5+ | OAuth集成 |
| **文件存储** | 对象存储 | - | S3兼容存储 |
| **AI能力** | LLM API + 知识库 | - | 流式输出、RAG检索 |
