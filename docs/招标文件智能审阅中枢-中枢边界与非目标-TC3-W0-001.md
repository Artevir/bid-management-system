# 中枢边界与非目标（TC3-W0-001）

**追溯**：《000.招标文件智能审阅中枢一步到位目标架构总装稿》口径摘录与工程化固定。  
**用途**：与 [V3.0 交付计划](./招标文件智能审阅中枢开发计划-交付级别-20260415-V3.0.md) 中 **TC3-W0-001** 对齐；**法律效力以 000 号 Word 为准**。

---

## 一、本期中枢负责（In Scope）

- 将**非结构化招标文件**加工为**可复用、可校验、可追溯、可联动**的招标侧结构化资产（对象化存储，**禁止**长期依赖单大 JSON 承载全量业务）。
- **接入、任务编排、规则治理、人工复核边界、状态与批次、溯源字段**等与 010–090 总装稿一致的能力域。
- 与**项目/文件/解读/标书侧既有模块**的衔接，以 **040 接口清单**与仓库实际 Route 对照表（附录 A）为执行参考。

---

## 二、本期中枢不负责（Out of Scope，至少三条）

1. **企业样式规范维护**、最终 Word **版式级装配**、整本标书「成稿级」导出（由下游模块承接，见 000）。
2. **保证金/履约保函/电子签/开标报价**等投标执行域业务（本仓库可有独立页面与 API，**不纳入**「智能审阅中枢」验收主链路，除非 300 冻结范围另行扩展）。
3. **客服/合作/样品/价格申请**等 support 域（附录 A 中标记为扩展域的页面/API，**不作为**中枢 P0 验收阻塞项，除非 040 将其纳入契约）。

---

## 三、冻结策略草案（TC3-W0-003 占位）

- **RC 后只增不改**：`src/db/schema` 中与「招标要求资产」主对象群相关的表、以及 **040 已公布**的公开 API 路径；变更须 ADR + 版本号（对齐 300）。
- **流程**：破坏性变更 → ADR → `CHANGELOG` → 迁移脚本 → 回归（160/290 子集）。

---

## 四、溯源字段最小集（TC3-W0-004 占位）

须与 **070** 总装稿逐字段对齐后在实现中落地；工程侧至少预留：原文锚点/页码或等价定位、批次 ID、规则版本、置信度、人工确认结果与操作者、时间戳。具体列名以 **070 Word** 为准。
