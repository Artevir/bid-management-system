**一、编制目的**

《220.招标文件智能审阅中枢可直接下发的第四批AI Coding提示词包（Scoring）-20260403-V1.0》用于在《190.招标文件智能审阅中枢可直接下发的首批AI Coding提示词包（Schema、Shared Types、Query、Bootstrap）-20260403-V1.0》已完成底座层、《200.招标文件智能审阅中枢可直接下发的第二批AI Coding提示词包（AI接入、任务编排、Ingestion、Requirements、Rules）-20260403-V1.0》已完成主链层、《210.招标文件智能审阅中枢可直接下发的第三批AI Coding提示词包（API、Layout、Overview、Requirements、Risks、Review）-20260403-V1.0》已完成第一阶段前后端闭环的前提下，进一步把第四批专题页扩展中的 **Scoring 专题包** 单独收成一份可直接复制下发的正式提示词执行稿，为后续真正落成评分专题页、评分详情、评分统计、证明材料提示、评分复核入口和页面联动提供统一作战底稿。

本文件只处理第四批中的 **Scoring**，不混入 Technical、Framework、Templates、Materials。这样做的目的，是把评分专题页作为一个独立专题做深、做稳、做可联调，而不是在一个大提示词中把多个专题页一起下发，导致边界失控、组件互相污染、状态逻辑混乱。

因此，本文件不是“第四批全部专题包”的泛泛说明，而是 **Scoring 专题页的直接作战文件**。

**二、文档定位**

本文件在整套“招标文件智能审阅中枢”方案中的定位，应明确为“第四批 Scoring 专题提示词总装章”。其上承 180 文档中的任务包体系，以及 190、200、210 三批提示词成果，其下直接约束以下内容：

一是约束 Scoring 专题页的改动目录、组件边界、状态边界和提示词正文。\
二是约束 Scoring 页面必须建立在既有 API、layout/common、Query、shared dto 和评分对象域之上。\
三是约束 Scoring 页面不得反向推翻前面三批已经确立的页面骨架、Query 口径和状态口径。\
四是约束评分专题页的重点，不是做花哨页面，而是把“评分结构、评分细项、证明材料提示、详情解释、原文溯源、动作入口”做成稳定页面。\
五是约束后续 Technical、Framework、Templates、Materials 专题页，也应沿用本文件对专题页提示词的组织方式。

如果没有本文件，后续虽然已有主链和基础页面，但在扩展 Scoring 专题页时仍然容易出现以下问题：

第一，评分页容易绕过既有 Query 和 API，直接在页面端拼统计。\
第二，评分页容易为了赶进度，复制 Requirements 页或 Risks 页的壳，却没有真正适配评分对象。\
第三，评分证明材料提示很容易只显示文本，不与评分项细节、来源页码和原文依据形成联动。\
第四，评分页的统计卡、筛选条件、详情区和动作区很容易风格跑偏。\
第五，后续再做 Technical、Framework 等专题页时，无法复用评分页沉淀出的专题页范式。

因此，本文件的目标，是把“Scoring 包怎么发、怎么做、做到什么算专题页成立”一次性定稳。

**三、Scoring 专题包选择原则**

Scoring 包必须同时满足以下五个条件：

- 分类结构

- 证明材料提示

Scoring 页面必须建立在：

- TenderCenterPageShell

- ProjectContextHeader

- PageTitleBar

- StandardSplitLayout

- StandardObjectTable

- StandardDetailPanel\
  等既有能力之上。

**3.4 必须具备专题页完整五要素**

Scoring 页面至少必须具备：

- 统计区

- 筛选区

- 列表区

- 详情区

- 原文/动作区

缺一不可。

**3.5 必须便于后续专题页复用**

Scoring 页面做完后，应能自然成为后续 Technical、Framework、Templates、Materials 专题页的参考范式。

**六、Scoring 页面设计目标**

Scoring 专题页应服务以下四类核心场景：

**6.1 快速看懂评分结构**

用户进入页面后，应能第一时间看到：

- 总分

- 商务分、技术分、价格分

- 评分项总数

- 证明材料相关评分项数

**6.2 快速筛选高价值评分项**

用户应能快速按以下条件筛选：

- 分类

- 分值区间

- 是否含证明材料提示

- reviewStatus

- 关键词

**6.3 快速查看评分细则详情**

用户点选一条评分项后，应能看到：

- 分类

- 条目名称

- 分值

- 评分标准全文

- 证明材料提示

- 来源页码/原文依据

**6.4 快速进入后续动作链**

评分页虽然不是复核主页，但至少应具备：

- 发起复核入口

- 定位原文入口

- 导出入口\
  这三个基础动作入口。
