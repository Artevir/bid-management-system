## 二、中枢定位（摘自 000 总装稿要点）

- **正式名称**：招标文件智能审阅中枢。
- **使命**：将非结构化招标文件，转化为可复用、可校验、可追溯、可联动、可驱动后续投标文件自动生成的 **标准化招标要求资产**（非单纯「解读结果展示」）。
- **边界**：当前阶段聚焦 **招标侧资产**；企业样式规范维护、最终 Word 版式装配、整本标书成稿导出等由下游模块承接（见 000 文）。
- **原则摘要**：主对象群拆分（非单一大 JSON）、内容与结构/模板/变量/风险/材料/复核分对象、全链路溯源字段、AI 与人工复核双轨（详见 000、070）。

## 五、里程碑与质量门禁（合并 160、170、300 口径）

1. **设计冻结**：010–080 对应对象、接口、页面、动作、规则、字段视图在评审会上签字或等同记录。
2. **可集成里程碑**：090–120 与后端路由、Schema 同步可运行；禁止「单分支长期不可运行」。
3. **数据与初始化门禁**：150 的 seed/示例数据可在干净库上一键还原并通过冒烟。
4. **联调与验收门禁**：160 清单 + 290 回归/演示/验收包全部通过后方可宣称阶段交付（见 300 冻结说明）。
5. **版本冻结**：300 发布后，对已冻结目录与接口的变更须走「增量扩展 / 版本升级」路径，避免静默破坏归档成果。

## 附录 A：Markdown 再生成方法

在仓库根目录执行：

```bash
python scripts/docx-hub-to-markdown.py
```

- 自动在 `docs` 下查找含 `000*.docx` 的目录（兼容目录名编码差异）。
- 跳过 Word 锁文件 `~$*.docx`。
- 输出至 **`docs/招标文件智能审阅中枢/`**（与 `.docx` 同名的 `.md` + `INDEX.md`）。
- **表格**：默认**优先 Pandoc** 输出 GFM；失败或未安装时回退内置 `w:tbl` 解析。强制仅用内置： **`USE_PANDOC_FOR_DOCX=0`**。
