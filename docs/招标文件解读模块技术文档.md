## 二、功能概述

招标文件解读模块是标书全流程管理平台的核心组件，主要功能包括：

1. **文档上传与解析** - 支持多种格式招标文件的智能解析
2. **结构化信息提取** - 从非结构化文档中提取关键信息
3. **多维度数据分析** - 技术规格、评分细则、时间节点等
4. **多格式导出** - 支持JSON、Excel、Word、TXT、PDF格式导出

    chapter: string;

  }>;
  
  // 时间节点

## 五、处理流程

### 5.1 文档解析流程

```
文件上传 → 文件类型识别 → 文本提取 → AI智能解析 → 结构化存储
```

1. **文件上传**: 用户上传招标文件（PDF/DOC/DOCX/XLS/XLSX）
2. **类型识别**: 根据文件扩展名选择对应的解析器
3. **文本提取**: 将文档内容转换为纯文本
4. **AI解析**: 调用大模型API提取关键信息
5. **结构化存储**: 将解析结果存入数据库

### 5.2 信息提取内容

| 提取模块 | 包含内容 |
|---------|----------|
| 基本信息 | 项目名称、编号、招标单位、预算等 |
| 费用信息 | 招标文件费、投标保证金、履约保证金等 |
| 提交要求 | 投标方式、截止时间、份数要求等 |
| 文档框架 | 章节结构、标题、页码等 |
| 时间节点 | 关键时间点、时间、地点等 |
| 技术规格 | 设备参数、技术要求等 |
| 评分细则 | 评分项目、分值、标准等 |
| 核对清单 | 资质要求、核验状态等 |

## 六、导出功能

### 6.1 支持的导出格式

| 格式 | 文件扩展名 | 内容结构 |
|------|-----------|----------|
| JSON | .json | 完整JSON对象 |
| Excel | .xlsx | 多工作表（按显示顺序排列） |
| Word | .docx | 带表格的Word文档 |
| TXT | .txt | 纯文本格式 |
| PDF | .pdf | PDF文档（需额外配置字体） |

### 6.2 Excel导出工作表顺序

1. **项目基本信息** - 基本信息字段
2. **费用相关信息** - 费用相关字段
3. **投标提交要求** - 提交要求列表
4. **文档框架** - 文档结构
5. **关键时间节点** - 时间节点列表
6. **技术规格要求** - 技术规格列表
7. **评分细则** - 评分项目列表
8. **资质要求核对清单** - 核对清单列表

### 6.3 导出API接口

```
GET /api/interpretations/[id]/export?format=[json|excel|word|txt|pdf]
```

**请求参数**:
- `format`: 导出格式（默认json）
- `modules`: 可选，指定导出模块（默认all）

**响应头**:
```http
Content-Type: [根据格式确定]
Content-Disposition: attachment; filename="{项目名}_解读结果_{日期}.{扩展名}"
```

## 八、注意事项

1. **中文字段处理**: 导出时字段名自动转换为中文，无需额外配置
2. **文件编码**: 所有文本导出使用UTF-8编码，确保中文显示正确
3. **大数据处理**: 对于大型文档采用分块处理，避免内存溢出
4. **错误处理**: 解析失败时记录详细日志，便于问题排查
5. **PDF导出**: 需要配置字体支持中文显示
