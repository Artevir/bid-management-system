# 系统修复报告-20260329-V1.10

**版本**: V1.10  
**日期**: 2026-03-29  

## 修复背景

服务器执行 `pnpm build` 在 TypeScript 类型校验阶段失败，阻断生产构建。

## Issue 1: `getLLM` 调用参数与函数签名不一致导致类型检查失败

* **所在文件**: `src/app/api/bid/chapters/[id]/generate/route.ts`
* **问题现象**:
  - 报错：`Expected 0 arguments, but got 1.`
  - 出错代码：`getLLM(forwardHeaders)`
* **根因分析**:
  - 当前 `getLLM()` 的签名为无参：`getLLM(): LLMAdapter`
  - 调用方仍按旧接口传入 `forwardHeaders`，导致 TS 类型检查失败
* **修复方案**:
  - 保留“需要时透传请求头给 Coze 适配器”的原设计意图
  - 通过 `LLMFactory.getInstance().getConfig().defaultProvider` 判断默认提供商：
    - 默认提供商为 `coze` 且存在可用 `forwardHeaders` 时，使用 `createCozeAdapterWithHeaders(forwardHeaders)`
    - 否则回退到 `getLLM()` 获取默认适配器
* **修复结果**:
  - 类型检查通过，不再阻断 `next build`
  - Coze 场景仍可按需透传鉴权/会话相关 Header
