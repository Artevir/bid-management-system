# 系统修复报告-20260329-V1.12

**版本**: V1.12  
**日期**: 2026-03-29  

## 修复背景

服务器执行 `pnpm build` 在 TypeScript 类型校验阶段失败，阻断生产构建。报错指向章节生成接口调用了不存在的流式方法。

## Issue 1: `LLMAdapter` 不包含 `chatStream`，调用方误用导致类型检查失败

* **所在文件**: `src/app/api/bid/chapters/[id]/generate/route.ts`
* **问题现象**:
  - 报错：`Property 'chatStream' does not exist on type 'LLMAdapter'.`
  - 出错代码：`llm.chatStream({ messages })`
* **根因分析**:
  - `LLMAdapter` 的接口定义位于 `src/lib/llm/types.ts`，不包含 `chatStream`
  - 仓库中已存在统一的流式对话函数：`streamChat(...)`（位于 `src/lib/llm/service.ts`）
  - 当前架构约定为：非流式调用走 `llm.chat(...)`，流式调用走 `streamChat(...)`
* **修复方案**:
  - 将流式分支从 `llm.chatStream(...)` 改为调用 `streamChat({ messages }, forwardHeaders)`
  - 保留 `forwardHeaders` 透传能力（用于 Coze 等需要鉴权/会话头的场景）
* **修复结果**:
  - 类型检查通过，不再阻断 `next build`
  - 章节流式输出逻辑按现有 LLM service 统一口径运行
