# 系统修复报告-20260329-V1.13

**版本**: V1.13  
**日期**: 2026-03-29  

## 修复背景

服务器执行 `pnpm build` 在 TypeScript 类型校验阶段失败，报错集中在章节生成接口对 LLM 调用方式不一致（适配器接口与调用方漂移），导致构建被阻断。

## Issue 1: 章节生成接口统一收口到 LLM service 层（流式/非流式）

* **所在文件**: `src/app/api/bid/chapters/[id]/generate/route.ts`
* **问题现象**:
  - 构建阶段出现类型错误：`LLMAdapter` 上不存在 `chatStream` / `chat`
  - 说明调用方仍在按旧接口直接调用适配器实例，但当前仓库 LLM 设计已将流式/非流式能力收口在 service 层
* **根因分析**:
  - `LLMAdapter` 的接口定义位于 `src/lib/llm/types.ts`，并不提供 `chatStream/chat`
  - 仓库已有稳定实现：
    - 流式：`streamChat(...)`（`src/lib/llm/service.ts`）
    - 非流式：`invokeChat(...)`（`src/lib/llm/service.ts`）
  - 章节生成接口混用了 “service 层” 与 “adapter 实例” 两套口径，引发类型不一致
* **修复方案**:
  - 章节生成接口不再直接调用适配器实例方法
  - 流式分支统一使用：`streamChat({ messages }, forwardHeaders)`
  - 非流式分支统一使用：`invokeChat({ messages }, forwardHeaders)`
  - 保留 `extractForwardHeaders(...)` 以在需要时透传鉴权/会话相关 Header
* **修复结果**:
  - LLM 调用口径统一，类型检查不再因该处阻断构建
  - 流式与非流式行为一致（均通过 service 层控制模型/温度等参数默认值）
