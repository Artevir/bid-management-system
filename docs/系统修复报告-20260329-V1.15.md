# 系统修复报告-20260329-V1.15

**版本**: V1.15  
**日期**: 2026-03-29  

## 修复背景

服务器执行 `pnpm build` 在 TypeScript 类型校验阶段失败，错误来自将异步权限中间件当作同步函数直接调用。

## Issue 1: `withDocumentPermission(...)` 为异步函数，需先 `await` 再调用

* **所在文件**: `src/app/api/bid/chapters/route.ts`
* **问题现象**:
  - 报错：`This expression is not callable. Type 'Promise<...>' has no call signatures.`
  - 出错代码：`withDocumentPermission('read', () => documentId)(request, ...)`
* **根因分析**:
  - `withDocumentPermission` 是 `async function`，返回类型为 `Promise<(request, handler, params?) => Promise<Response>>`
  - 调用方将返回的 `Promise` 直接当函数调用，导致类型检查失败
* **修复方案**:
  - 改为两步调用：
    - `const middleware = await withDocumentPermission(...)`
    - `return middleware(request, handler)`
  - GET/POST 分别使用 `readMiddleware` / `editMiddleware`，与项目内 `withChapterPermission` 的正确用法保持一致
* **修复结果**:
  - 章节列表与章节创建 API 路由类型检查通过
  - 该类“Promise 当函数调用”的错误不再阻断构建
