# 系统修复报告-20260329-V1.16

**版本**: V1.16  
**日期**: 2026-03-29  

## 修复背景

服务器执行 `pnpm build` 在 TypeScript 类型校验阶段失败，错误出现在“永久删除文档”接口调用回收站服务函数的参数签名不一致。

## Issue 1: `permanentDelete` / `moveToRecycleBin` 调用参数与服务签名不一致

* **涉及文件**:
  - `src/app/api/bid/documents/[id]/route.ts`
  - `src/lib/recycle-bin/service.ts`
* **问题现象**:
  - 报错：`Expected 2 arguments, but got 3.`
  - 出错代码：`permanentDelete('document', documentId, userId)`
* **根因分析**:
  - 回收站服务层函数签名为：
    - `moveToRecycleBin(params: MoveToRecycleBinParams)`
    - `permanentDelete(recycleBinId: number, deletedBy: number)`
  - 调用方仍使用旧式签名（按 `resourceType/resourceId/userId` 传参），导致类型不匹配
* **修复方案**:
  - 文档删除接口改为先调用 `moveToRecycleBin({ resourceType, resourceId, deletedBy })` 获取 `recycleBinId`，再调用 `permanentDelete(recycleBinId, userId)` 完成物理删除
  - 同时调整 `moveToRecycleBin` 的幂等行为：当资源已在回收站中时，返回 `success: true` 并带回已存在的 `recycleBinId`，便于“永久删除”链路复用
* **修复结果**:
  - `next build` 类型检查阶段不再被该处阻断
  - “移至回收站”与“永久删除”链路与 service 层签名一致，并具备幂等性
