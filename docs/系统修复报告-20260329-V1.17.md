# 系统修复报告-20260329-V1.17

**版本**: V1.17  
**日期**: 2026-03-29  

## 修复背景

服务器执行 `pnpm build` 在 TypeScript 类型校验阶段失败，错误出现在缓存管理接口对“项目/用户缓存失效函数”的调用参数类型不一致。

## Issue 1: Redis 缓存失效函数与内存缓存失效函数同名导致类型不匹配

* **所在文件**: `src/app/api/cache/route.ts`
* **问题现象**:
  - 报错：`Argument of type 'string' is not assignable to parameter of type 'number'.`
  - 出错代码：`invalidateProjectCache(projectId)`（`projectId` 来自 query string）
* **根因分析**:
  - `@/lib/cache` 同时存在两套失效函数：
    - 内存缓存：`invalidateProjectCache(projectId: number)` / `invalidateUserCache(userId: number)`
    - Redis 客户端：`invalidateProjectCache(projectId: string)` / `invalidateUserCache(userId: string)`
  - barrel 文件对 Redis 版本采用了重命名导出：
    - `invalidateProjectRedisCache`
    - `invalidateUserRedisCache`
  - 该 API 属于 Redis 缓存管理接口，但调用方误导入了“内存缓存”版本，导致参数类型冲突
* **修复方案**:
  - 将项目/用户缓存失效调用统一切到 Redis 版本导出：
    - `invalidateProjectRedisCache(projectId)`
    - `invalidateUserRedisCache(userId)`
* **修复结果**:
  - `next build` 类型检查阶段不再被该处阻断
  - 缓存管理接口的语义与实现对齐（按 Redis key pattern 进行失效）
