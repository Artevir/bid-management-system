# 系统修复报告-20260329-V1.20

**版本**: V1.20  
**日期**: 2026-03-29  

## 修复背景

服务器执行 `pnpm build` 在 TypeScript 类型校验阶段失败，错误发生在导出接口将 Node.js `Buffer` 作为 `Blob` 的 part 传入时，类型不兼容。

## Issue 1: `Buffer<ArrayBufferLike>` 不满足 `BlobPart`（SharedArrayBuffer 类型差异）

* **所在文件**: `src/app/api/export/route.ts`
* **问题现象**:
  - 报错：`Type 'Buffer<ArrayBufferLike>' is not assignable to type 'BlobPart'.`
  - 具体原因：`Buffer.buffer` 的类型为 `ArrayBufferLike`，可能为 `SharedArrayBuffer`，而当前 `BlobPart` 类型要求 `ArrayBuffer`/`ArrayBufferView<ArrayBuffer>`。
* **根因分析**:
  - 构建环境的 lib 类型定义中，将 `SharedArrayBuffer` 与 `ArrayBuffer` 区分得更严格
  - 直接 `new Blob([buffer])` 会触发 `Buffer` 与 `BlobPart` 的类型不匹配
* **修复方案**:
  - 将 `Buffer` 拷贝为标准 `Uint8Array<ArrayBuffer>`，再作为 `Blob` part：
    - `const bytes = Uint8Array.from(buffer)`
    - `const body = new Blob([bytes], { type: mimeType })`
* **修复结果**:
  - `next build` 类型检查阶段不再被该处阻断
  - 导出接口继续返回二进制文件下载（headers 维持不变）
