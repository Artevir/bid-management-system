# 系统修复报告-20260329-V1.24

**版本**: V1.24  
**日期**: 2026-03-29  

## 修复背景

服务器执行 `pnpm type-check`（`tsc --noEmit`）一次性暴露出多处 TypeScript 类型问题，分布在前端组件、权限中间件、搜索/报表服务、调度器、对象存储与测试用例中。本轮按“批量收口同类问题”的方式集中修复，减少后续逐条阻断式返工。

## 修复内容汇总

### 1) 前端组件：缺失图标导入 / Props 漂移

* **文件**:
  - `src/components/bid/document-overview-panel.tsx`
  - `src/components/bid/editor/editor-content.tsx`
  - `src/components/editor/ai-inline-editor.tsx`
* **修复点**:
  - 补齐 `AlertCircle` 从 `lucide-react` 的导入
  - `AIInlineEditor` 组件 Props 以 `value` 为主，调用方从 `content` 调整为 `value`
  - 为保持调用方兼容，`AIInlineEditor` 增加可选 `streamingContent?: string`

### 2) 公司组件导出：补齐类型导出

* **文件**:
  - `src/components/company/company-form.tsx`
* **修复点**:
  - 补充 `export type { CompanyFormData }`，使 `src/components/company/index.ts` 的 re-export 正常

### 3) 权限/中间件：统一返回 `Response` 契约

* **文件**:
  - `src/lib/auth/project-middleware.ts`
* **修复点**:
  - 由于 `withAuth` 已收口为 `Response` 返回，项目级中间件将 handler/返回类型从 `Promise<NextResponse>` 放宽为 `Promise<Response>`，避免再次出现 `Response` 与 `NextResponse` 的类型不兼容

### 4) 审批/业务服务：缺失 schema 引用与错误码类型

* **文件**:
  - `src/lib/bid/approval.ts`
  - `src/lib/bid/documents-service.ts`
* **修复点**:
  - 补齐 `projects` 的 schema 导入，修复 `Cannot find name 'projects'`
  - 将乐观锁冲突错误由 `new AppError(..., 409)` 调整为 `AppError.conflict(...)`，与 `ErrorCode` 枚举契约一致

### 5) 报表/搜索：schema 字段对齐 + 结果类型稳定

* **文件**:
  - `src/lib/analytics/report-service.ts`
  - `src/lib/search/search-service.ts`
* **修复点**:
  - `projects` 表不存在 `companyId` 字段，相关统计与过滤改为使用 `projects.platformId`
  - `SearchResult.id` 放宽为 `string | number`，避免 number/string 漂移导致的集合类型不兼容
  - `users` 表不存在 `name` 字段，改为使用 `realName/username`
  - `highlightText` 参数按需将 `null` 归一为 `undefined`

### 6) 基础设施：索引管理/加密装饰器/调度器/对象存储类型修复

* **文件**:
  - `src/lib/db/index-manager.ts`
  - `src/lib/encryption/encryption-service.ts`
  - `src/lib/scheduler/scheduler-service.ts`
  - `src/lib/storage/oss-client.ts`
* **修复点**:
  - 索引管理：补齐 `implicit any`，并将查询结果显式映射为 `IndexInfo[]`，避免不安全的强制转换
  - 加密装饰器：递归函数补充返回类型标注，消除递归推断导致的隐式 `any`
  - 调度器：使用 `node-cron` 导出的 `ScheduledTask` 类型替代不存在的 `cron` namespace，并移除不被接受的 `scheduled` 选项字段
  - OSS 上传：用 `Buffer.isBuffer` 进行类型收窄，避免在 `Buffer | File` 联合类型上访问 `arrayBuffer/size/name`

### 7) 测试用例：修正不存在的 schema 路径

* **文件**:
  - `tests/unit/project.test.ts`
* **修复点**:
  - `@/db/schema/projects` 不存在，修正为从 `@/db/schema` 导入 `projects`
