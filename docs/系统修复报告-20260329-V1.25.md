# 系统修复报告-20260329-V1.25

**版本**: V1.25  
**日期**: 2026-03-29  

## 修复背景

服务器执行 `pnpm type-check` / `pnpm build` 阻断，错误集中在文档导出 API：同一作用域重复声明变量 `body`，以及 `instanceof Uint8Array` 在联合类型下不满足 TypeScript 约束。

## Issue 1: `body` 变量重复声明导致编译/打包失败

* **所在文件**: `src/app/api/bid/export/route.ts`
* **问题现象**:
  - TS 报错：`Cannot redeclare block-scoped variable 'body'.`
  - Webpack 报错：`Identifier 'body' has already been declared`
* **根因分析**:
  - `handleExport` 内既用 `const body = await request.json()` 表示请求体
  - 又用 `const body = new Blob(...)` 表示响应 body，发生同作用域重复声明
* **修复方案**:
  - 将请求体变量更名为 `payload`
  - 将响应体变量更名为 `responseBody`

## Issue 2: `instanceof Uint8Array` 在联合类型下触发 TS2358

* **所在文件**: `src/app/api/bid/export/route.ts`
* **问题现象**:
  - TS 报错：`The left-hand side of an 'instanceof' expression must be of type 'any', an object type or a type parameter.`
* **根因分析**:
  - `result.content` 的静态类型包含非对象分支（例如字符串），直接 `result.content instanceof Uint8Array` 触发 TS2358
* **修复方案**:
  - 使用安全的类型收窄写法：`(result.content as any) instanceof Uint8Array`
  - 维持 `Buffer.isBuffer` 的判断不变

## 修复结果

- `pnpm type-check` / `pnpm build` 不再在该文件处阻断
- 导出接口依旧以 `Blob` 形式返回二进制内容，下载行为保持一致
