# 系统修复报告-20260329-V1.26

**版本**: V1.26  
**日期**: 2026-03-29  

## 修复背景

服务器执行 `pnpm build` 在 “Collecting page data ...” 阶段失败，错误为 Next.js 服务器端 bundle 运行时缺失 chunk 文件（`.next/server/webpack-runtime.js` 尝试 `require("./5351.js")` 但文件未生成/路径不一致）。

## Issue 1: Next.js server runtime 期望的 chunk 输出路径与实际产物不一致

* **涉及文件**: `next.config.ts`
* **问题现象**:
  - `unhandledRejection Error: Cannot find module './5351.js'`
  - require stack 指向 `.next/server/webpack-runtime.js` → `.next/server/pages/_document.js`，说明 server runtime 在加载 server chunk 时找不到对应文件
* **根因分析**:
  - Next.js 在 page data 收集阶段会直接 `require` server bundle
  - 当 server 侧 chunk 文件名/落盘路径与 runtime 生成的 `require("./<id>.js")` 不一致时，会在该阶段直接阻断构建
* **修复方案**:
  - 在 server 构建时显式收口 webpack server chunk 输出命名，确保 chunk 以 `<id>.js` 形式落在 `.next/server/` 根目录，匹配 runtime 的 require 路径：
    - `config.output.chunkFilename = '[id].js'`
* **修复结果**:
  - 预期可消除 “Collecting page data” 阶段因缺失 `./<id>.js` 而阻断的构建失败

## 验证建议（服务器）

```bash
rm -rf .next
pnpm build
```
