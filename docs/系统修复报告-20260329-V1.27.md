# 系统修复报告-20260329-V1.27

**版本**: V1.27  
**日期**: 2026-03-29  

## 修复背景

服务器执行 `pnpm build` 在预渲染阶段失败（`Error occurred prerendering page "/bid"`、`"/approval"`），报错为：`No QueryClient set, use QueryClientProvider to set one`。

## Issue 1: React Query 未在应用根部注入 `QueryClientProvider`，导致预渲染时抛错

* **涉及页面**:
  - `/bid`
  - `/approval`
* **问题现象**:
  - 构建阶段预渲染报错：`No QueryClient set, use QueryClientProvider to set one`
* **根因分析**:
  - 页面为 Client Component（`'use client'`），内部调用了 `@tanstack/react-query` 的 hooks（如 `useQuery` / `useQueryClient`）
  - 应用根布局未包裹 `QueryClientProvider`，导致在预渲染/服务端渲染阶段执行 hooks 时抛错
* **修复方案**:
  - 新增全局 provider：`src/components/providers/react-query-provider.tsx`
  - 在 `src/app/layout.tsx` 根布局包裹 `ReactQueryProvider`，确保所有页面/组件均能拿到 `QueryClient`
* **修复结果**:
  - `pnpm build` 预渲染阶段不再因 QueryClient 缺失而阻断

## 验证建议（服务器）

```bash
rm -rf .next
pnpm build
```
