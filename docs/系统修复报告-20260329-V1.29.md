# 系统修复报告-20260329-V1.29

**版本**: V1.29  
**日期**: 2026-03-29  

## 修复背景

生产环境登录后出现白屏并提示 `Minified React error #31`。该错误的完整含义是：组件渲染时把“对象”当作 React 子节点渲染（`Objects are not valid as a React child`）。

## Issue 1: 登录失败分支将对象写入 `error` state，导致渲染崩溃

* **所在文件**: `src/app/login/page.tsx`
* **问题现象**:
  - 生产环境登录失败（例如用户名/密码错误）后页面崩溃
  - React 报错：`Minified React error #31`（对象被当作 React child 渲染）
* **根因分析**:
  - 登录失败时逻辑为：
    - `setError(data.error || '登录失败')`
  - 若后端返回结构为：
    - `{ error: { code: 'INVALID_CREDENTIALS', message: '用户名或密码错误' } }`
  - 则 `data.error` 为对象，被写入 `error` state
  - 页面渲染处：
    - `<AlertDescription>{error}</AlertDescription>`
  - 导致 React 尝试渲染对象，触发 error #31
* **修复方案**:
  - 登录失败时将错误信息统一归一为字符串，再写入 state：
    - `typeof data?.error === 'string' ? data.error : typeof data?.error?.message === 'string' ? data.error.message : ...`
  - 对 `response.json()` 增加解析保护，避免非 JSON 响应导致二次崩溃
  - 渲染层做双保险：`<AlertDescription>{String(error)}</AlertDescription>`
* **修复结果**:
  - 登录失败时能正常显示错误文本，不再触发 React error #31
  - 兼容后端返回 `error` 为字符串或对象（`{ code, message }`）两种格式
