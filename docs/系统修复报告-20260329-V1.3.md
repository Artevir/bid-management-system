# 系统修复报告-20260329-V1.3

**版本**: V1.3  
**日期**: 2026-03-29  

## 修复背景

服务器执行 `pnpm build` 时，Next.js 在编译阶段报错：

`Module parse failed: Export 'redis' is not defined`

错误发生在 `src/lib/cache/index.ts` 的导出区域，导致构建被阻断。

## Issue 1: `src/lib/cache/index.ts` 重新导出方式错误导致构建失败

* **所在文件**: `src/lib/cache/index.ts`
* **问题现象**:
  - 构建报 `Export 'redis' is not defined`
  - 报错定位到 `export { redis as cache };` / `export default redis;`
* **根因分析**:
  - `index.ts` 仅做了 `export { redis, ... } from './redis-client'` 的 re-export
  - 但随后使用了 `export { redis as cache }; export default redis;`
  - 在 ES Module 语义下，re-export 并不会在当前模块创建本地绑定变量，因此 `redis` 在当前文件作用域中并不存在，Webpack 解析时报错。
* **修复方案**:
  - 改为“直接从源模块重新导出”，避免依赖当前文件的本地绑定：
    - `export { redis as cache } from './redis-client';`
    - `export { default } from './redis-client';`
* **修复结果**:
  - `@/lib/cache` 同时支持：
    - `import { cache } from '@/lib/cache'`
    - `import redis from '@/lib/cache'`
    - `import { redis, RedisKeys, getRedisClient, ... } from '@/lib/cache'`
  - 构建阶段不再出现该模块导出错误。
