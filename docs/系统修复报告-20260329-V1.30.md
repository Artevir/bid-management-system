# 系统修复报告-20260329-V1.30

**版本**: V1.30  
**日期**: 2026-03-29  

## 修复背景

输入错误密码后仍出现 `Minified React error #31`。该错误的完整含义为：React 渲染阶段把对象当作子节点渲染（`Objects are not valid as a React child`），常见触发点包括把对象直接传给 Toast/Alert 等 UI 组件。

## Issue 1: API 客户端全局错误处理将对象作为 toast 内容导致渲染崩溃

* **所在文件**: `src/lib/api/client.ts`
* **问题现象**:
  - 当后端返回错误结构为 `{ error: { code, message } }` 时
  - 全局错误处理逻辑直接使用 `data.error` 作为错误消息来源，可能将对象传给 `toast.error(...)`
  - UI 层渲染 toast 时触发 React error #31
* **根因分析**:
  - `data.error || data.message` 在 `data.error` 为对象时会返回对象
  - `toast.error(...)` 期望可渲染的内容（字符串/ReactNode），对象会导致渲染报错
* **修复方案**:
  - 在 API 客户端中将错误信息统一归一为字符串（优先级：`error(string)` → `error.message` → `message` → fallback）
  - 同时为 `response.json()` 增加解析保护，避免非 JSON 响应导致二次异常
* **修复结果**:
  - 全局 toast 不再因错误对象导致 React error #31
  - 与登录页修复形成闭环：保证错误信息在进入 UI 层前已被字符串化
