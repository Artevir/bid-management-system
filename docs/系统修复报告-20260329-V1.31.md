# 系统修复报告-20260329-V1.31

**版本**: V1.31  
**日期**: 2026-03-29  

## 修复背景

在生产环境输入错误密码时仍触发 `Minified React error #31`，错误含义为：React 渲染阶段把对象当作子节点渲染（`Objects are not valid as a React child (found: object with keys {code, message})`）。

## Issue 1: UI 组件直接渲染错误对象导致崩溃（防御性修复）

* **所在文件**: `src/components/ui/alert.tsx`
* **问题现象**:
  - 某些页面/组件在错误场景下可能把后端的错误对象 `{ code, message }` 直接传给 `AlertDescription` 作为 children
  - React 在渲染时会抛出 error #31 并导致页面白屏
* **修复方案**:
  - 为 `AlertDescription` 增加 children 归一逻辑：
    - 若 children 为对象且包含 `message: string`，优先渲染 `message`
    - 若 children 为普通对象，兜底使用 `String(children)`，避免 React 直接渲染对象崩溃
* **修复结果**:
  - 即使上层误把 `{ code, message }` 作为 children 传入，也不会再触发 React error #31
  - 页面可继续显示错误文本并保持可用
