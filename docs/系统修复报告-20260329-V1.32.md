# 系统修复报告-20260329-V1.32

**版本**: V1.32  
**日期**: 2026-03-29  

## 修复背景

在登录失败场景下仍出现 `Minified React error #31`，该错误表示 React 渲染阶段把“对象”当作子节点渲染（常见形态为 `{ code, message }`）。

## Issue 1: 登录页 `response.json()` 未保护 + 渲染侧统一字符串化

* **所在文件**: `src/app/login/page.tsx`
* **修复点**:
  - 为 `response.json()` 增加解析保护，避免非 JSON 响应导致异常分支失控
  - `AlertDescription` 渲染时统一 `String(error)`，保证子节点稳定为字符串

## Issue 2: Sonner toast 防御性补丁，避免对象消息触发 React #31

* **涉及文件**:
  - `src/components/providers/sonner-patch-provider.tsx`
  - `src/app/layout.tsx`
* **修复点**:
  - 在客户端启动时对 `toast.error/success/info/warning/...` 做统一 message 归一（对象优先取 `message`，否则 JSON 化）
  - 在根布局挂载 `Toaster`，确保 toast 渲染入口存在，并使补丁对全站生效
* **修复结果**:
  - 即使上层调用 `toast.error(result.error)` 且 `result.error` 为 `{ code, message }`，也不会再触发 React #31
