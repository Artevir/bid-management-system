# 系统修复报告-20260329-V1.33

**版本**: V1.33  
**日期**: 2026-03-29  

## 修复背景

在生产环境中仍出现 `Minified React error #31`（对象被当作 React 子节点渲染）。尽管登录页已将错误信息归一为字符串，但项目内大量代码仍可能将 `{ code, message }` 形式的错误对象直接传递给 Sonner 的 `toast.*`，从而在 toast 渲染阶段触发同类崩溃。

## Issue 1: Sonner toast 归一补丁在 `useEffect` 阶段执行过晚

* **涉及文件**: `src/components/providers/sonner-patch-provider.tsx`
* **问题现象**:
  - 若错误 toast 在首屏交互或其他 effect 链路中较早触发，可能先于 `SonnerPatchProvider` 的 `useEffect` 执行
  - 造成 toast 仍接收到对象 message 并触发 React #31
* **修复方案**:
  - 将 toast 归一补丁改为“模块加载即执行”的方式（客户端模块 import 时立即 patch），确保任何 toast 调用之前已经完成归一
* **修复结果**:
  - `toast.error/success/info/warning/...` 在全站范围内强制将对象 message 归一为可渲染字符串（优先取 `.message`，否则字符串化/JSON 化）
