# 系统修复报告-20260329-V1.35

**版本**: V1.35  
**日期**: 2026-03-29  

## 修复背景

引入 toast 相关逻辑后出现“服务器内部错误”。该类错误常见原因是：在服务端执行路径（SSR/Server Components/Node 运行时）中意外加载了仅浏览器可用的 UI 依赖（例如 Sonner），导致运行时崩溃并表现为 500。

## Issue 1: API 客户端在服务端路径被加载时不应静态依赖 `sonner`

* **所在文件**: `src/lib/api/client.ts`
* **问题现象**:
  - 若 `client.ts` 被服务端代码路径引用，静态 `import { toast } from 'sonner'` 可能在无浏览器对象的环境中触发异常
* **修复方案**:
  - 移除静态 `sonner` 依赖
  - 在需要 toast 的分支中，改为仅在浏览器环境（`typeof window !== 'undefined'`）下动态 `import('sonner')` 再调用 `toast.error`
* **修复结果**:
  - 服务端执行路径不再因加载 UI 依赖导致 500
  - 客户端仍保留原有错误 toast 行为

## Issue 2: 根布局移除 toast 补丁 Provider，减少服务端/客户端边界不确定性

* **所在文件**: `src/app/layout.tsx`
* **修复点**:
  - 移除 `SonnerPatchProvider` 挂载，仅保留 `Toaster` 作为 toast 渲染入口
