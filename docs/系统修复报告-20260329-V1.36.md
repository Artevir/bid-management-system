# 系统修复报告-20260329-V1.36

**版本**: V1.36  
**日期**: 2026-03-29  

## 修复背景

生产环境存在两类高频问题：

1. 登录/列表/表单提交失败后出现 `Minified React error #31`，其完整含义为：React 渲染阶段把对象当作子节点渲染（例如 `{ code, message }`）。
2. 构建或运行阶段出现 503/服务器内部错误，表现为页面不可用或 `next build` 在预渲染阶段退出。

本轮针对第 1 类问题做“全项目收口”，目标是将错误信息在进入 UI 层之前统一归一为字符串，彻底消除对象作为 React child 的可能性。

## Issue 1: 错误对象被当作 React 子节点渲染，触发 React error #31（全项目）

### 问题现象

在多个页面/组件中存在如下模式：

- `setError(data.error || 'xxx')`
- `toast.error(result.error || 'xxx')`
- `throw new Error(data.error || 'xxx')`
- `<AlertDescription>{error}</AlertDescription>`

当后端返回结构为：

```json
{ "error": { "code": "INVALID_CREDENTIALS", "message": "用户名或密码错误" } }
```

上述逻辑会把对象传入 state/toast/error message，最终在渲染层触发：

`Objects are not valid as a React child (found: object with keys {code, message})`

### 修复方案

1. 新增统一错误信息提取函数：将 `unknown` 归一为可渲染字符串（优先级：`error(string)` → `error.message` → `message` → JSON 字符串化 → fallback）。
2. 全仓替换高风险用法：
   - 所有 `setError(xxx.error || ...)` 改为 `setError(extractErrorMessage(xxx, fallback))`
   - 所有 `toast.error(xxx.error || ...)` 改为 `toast.error(extractErrorMessage(xxx, fallback))`
   - 所有 `throw new Error(xxx.error || ...)` 改为 `throw new Error(extractErrorMessage(xxx, fallback))`
   - 所有 `<AlertDescription>{error}</AlertDescription>` 改为 `<AlertDescription>{String(error)}</AlertDescription>`

### 关键改动

- 新增文件：`src/lib/error-message.ts`
- 登录页：`src/app/login/page.tsx`
- 多个业务页面/组件：将错误展示与 toast 全部改为统一归一方式（覆盖：保证金/开标/项目/回收站/厂家支持/模板分类/图片生成/任务规划等）

### 修复结果

- 后端即使返回 `{ code, message }` 形式的错误对象，也不会再作为 React child 进入渲染链路
- UI 层稳定展示字符串错误信息，不再出现 React error #31 白屏

## Issue 2: build 预渲染报错 `No QueryClient set`（构建失败/服务异常）

### 问题现象

构建阶段出现：

`Error: No QueryClient set, use QueryClientProvider to set one`

导致 `next build` 退出，从而引发运行端 503 或无法更新构建产物。

### 修复方案

确保根布局在应用树最外层提供 `QueryClientProvider`（通过 `ReactQueryProvider` 统一挂载）。

### 验证建议（服务器）

```bash
rm -rf .next
pnpm build
pm2 restart bid-management-system
```

验证点：

- 随便输入错误账号/密码：页面应展示错误文本，不再出现 React error #31
- `/api/health` 返回 200（若返回 503，需结合 `pm2 logs bid-management-system` 定位数据库/缓存/运行时异常）
