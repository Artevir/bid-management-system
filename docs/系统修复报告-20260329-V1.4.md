# 系统修复报告-20260329-V1.4

**版本**: V1.4  
**日期**: 2026-03-29  

## 修复背景

服务器 `pnpm build` 在 “Checking validity of types” 阶段失败，属于前端 TypeScript 类型推断问题，导致生产构建被阻断。

## Issue 1: `useState` 初始化空数组导致 `never[]` 推断，进而触发类型错误

* **所在文件**: `src/app/(app)/bid/[id]/edit/page.tsx`
* **问题现象**:
  - 构建报错：`Type '{ id: number; name: string; }' is not assignable to type 'never'.`
  - 报错位置：`setOptions({ templates: [{ id: 1, name: '通用投标响应模板' }], ... })`
* **根因分析**:
  - `useState({ templates: [], companies: [], availableTags: [] })` 中的空数组在缺少显式类型时会被推断为 `never[]`
  - 后续给 `templates` 赋值为对象数组时，TypeScript 判定无法赋值给 `never[]` 从而阻断编译
* **修复方案**:
  - 为 options state 增加明确类型定义：
    - `OptionItem`（`{ id: number; name: string }`）
    - `OptionsState`（包含 `templates/companies/availableTags` 三个数组）
  - 对 API 返回的 `companiesRes/tagsRes` 做最小字段映射，确保写入 state 的数据满足 `OptionItem[]` 结构
* **修复结果**:
  - `templates` 等字段不再被推断为 `never[]`
  - `next build` 的类型检查阶段不再被该处阻断
