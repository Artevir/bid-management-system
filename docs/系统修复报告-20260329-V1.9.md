# 系统修复报告-20260329-V1.9

**版本**: V1.9  
**日期**: 2026-03-29  

## 修复背景

在 Next.js 15.1.3 下执行 `pnpm build` 的类型检查阶段，认证登录接口因使用不存在的 `NextRequest.ip` 属性导致编译失败。

## Issue 1: NextRequest 不支持 `ip` 属性导致类型检查失败

* **所在文件**: `src/app/api/auth/login/route.ts`
* **问题现象**:
  - 报错：`Property 'ip' does not exist on type 'NextRequest'.`
  - 出错代码：`request.ip || 'unknown'`
* **根因分析**:
  - 在当前 Next.js / NextRequest 类型定义中不提供 `request.ip` 字段
  - 获取客户端 IP 应从反向代理/边缘网关注入的 Header 中读取
* **修复方案**:
  - 移除 `request.ip` 读取逻辑
  - 按优先级从 Header 获取：
    - `x-forwarded-for`（取第一个 IP）
    - `x-real-ip`
    - `cf-connecting-ip`（Cloudflare 场景）
  - 兜底返回 `'unknown'`
* **修复结果**:
  - 登录接口类型检查通过
  - 生产环境通过反代头仍可获得真实客户端 IP（或至少可追踪到网关 IP）
